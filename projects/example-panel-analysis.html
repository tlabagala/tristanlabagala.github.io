The question

Does the density of police stations across local government areas (LGAs) predict property crime rates, once we account for everything about an area that does not change over time? This example uses simulated data to demonstrate the analysis structure; the real project would use the Victorian LGA crime panel.

Data and cleaning

A balanced panel of 79 simulated LGAs observed annually from 2014 to 2023. In a real version the work is mostly in the cleaning: harmonising LGA names across sources, resolving a composite key of area by year, and dropping all-zero rows before the join. Here the data is generated below so the page is self-contained.

Code 
library(tidyverse)
source(here::here("theme/theme_portfolio.R"))

set.seed(42)
area_types <- c("Inner metro", "Outer metro", "Regional", "Rural")

panel <- expand_grid(area = 1:79, year = 2014:2023) |>
  mutate(
    type = area_types[1 + (area %% 4)],
    station_density = rnorm(n(), 2, 0.5),
    # true effect only in outer-metro areas
    effect = if_else(type == "Outer metro", -0.9, -0.1),
    crime_rate = 60 + effect * station_density +
      rnorm(n(), 0, 3) + as.numeric(factor(area)) * 0.05
  )

Method, and why this one

Two-way fixed effects: area effects absorb anything constant within an LGA (baseline affluence, geography), and year effects absorb shocks common to all areas in a year (state-wide policy). The estimate then rests on within-area change over time, not on comparing one area to another.

Why this rather than pooled OLS? Pooled OLS would let stable differences between areas leak into the coefficient. Fixed effects is the standard defence against that time-invariant confounding.

Two-way fixed effects absorbs anything constant within an LGA, so the estimate rests on within-area change over time - not on comparing Melton to Melbourne.

Code 
library(fixest)

m <- feols(crime_rate ~ station_density | area + year, data = panel)

# effect estimated separately by area type, for the figure
by_type <- panel |>
  group_split(type) |>
  map_dfr(\(d) {
    fit <- feols(crime_rate ~ station_density | area + year, data = d)
    tibble(type = unique(d$type),
           estimate = coef(fit)[["station_density"]],
           se = se(fit)[["station_density"]])
  })

The pooled coefficient on station density is -0.44. Broken out by area type, the effect concentrates in outer-metro areas and is negligible elsewhere.

Assumptions and limitations

This design assumes that, after removing area and year effects, station density is not driven by an omitted time-varying factor. That assumption is doing a lot of work: if areas add stations in response to rising crime, the coefficient is biased regardless of the fixed effects. The standard errors here are classical; a real analysis would cluster by area to allow for serial correlation within an LGA. And because the data is simulated, none of these numbers say anything about the world - they only demonstrate the pipeline.
